# 웹/앱 기획서 좋은 예시 모음

## 예시 1: Lunch Pick (점심 메뉴 추천 앱)

### 1. 프로젝트 개요
- **프로젝트명**: Lunch Pick (오늘의 점심)
- **한줄 요약**: 직장인을 위한 AI 점심 메뉴 추천 서비스
- **유형**: 신규 개발 (MVP)
- **예상 기간**: 3개월 (2025.01 ~ 2025.03)

### 2. 문제 정의 및 해결책

#### 현황 분석
- 직장인 78%가 매일 점심 메뉴 결정에 고민 (출처: 2024 직장인 식사 실태조사)
- 메뉴 결정에 평균 15분 소요
- 팀원 간 의견 충돌로 스트레스 발생률 45%

#### Pain Point
1. **결정 피로**: 매일 반복되는 "뭐 먹지?" 고민
2. **메뉴 고착화**: 같은 메뉴만 반복, 영양 불균형
3. **그룹 조율 어려움**: 5인 이상 팀 점심 시 의견 조율 평균 20분

#### 해결책
- **개인화 추천**: 취향/히스토리 기반 맞춤 메뉴 3개 제시
- **그룹 투표**: 팀원 선호도 종합하여 최적 메뉴 도출
- **다양성 보장**: 최근 3일 내 먹은 메뉴 자동 제외

#### Why Now
- 코로나 이후 오피스 복귀로 단체 식사 수요 급증 (전년 대비 +35%)
- ChatGPT 등 AI 추천 기술 대중화
- MZ세대 "결정 피로(Decision Fatigue)" 해소 니즈

### 3. 타겟 사용자

#### Primary Target: 김대리 페르소나
- **프로필**: 32세, IT기업 마케터, 서울 강남
- **상황**: 10명 규모 팀, 주 3회 팀 점심
- **Pain Point**: "매번 점심 메뉴 정하는 게 제일 스트레스"
- **기대 가치**: 1분 내 결정, 팀원 모두 만족
- **사용 빈도**: 주 5회

#### Secondary Target: 박팀장 페르소나
- **프로필**: 42세, 스타트업 팀장, 서울 역삼
- **상황**: 8명 팀 리드, 신입 온보딩 담당
- **Pain Point**: 신입에게 식당 추천 부담
- **기대 가치**: 상황별 맞춤 식당 추천

#### 시장 규모
- **TAM**: 국내 직장인 2,000만명
- **SAM**: 수도권 화이트칼라 500만명
- **SOM**: IT/스타트업 밀집 지역 50만명 → 1년차 5만명 목표

### 4. 핵심 기능

#### Must Have (MVP)
| 기능 | 설명 | User Story |
|------|------|------------|
| 소셜 로그인 | 카카오/구글 원터치 가입 | 사용자는 3초 내에 가입할 수 있다 |
| 선호도 설정 | 음식 종류, 맵기, 알러지 | 사용자는 취향을 5분 내 설정할 수 있다 |
| 오늘의 추천 | AI 기반 3개 메뉴 추천 | 사용자는 1분 내 점심을 결정할 수 있다 |
| 식당 지도 | 추천 메뉴 판매 식당 표시 | 사용자는 가까운 식당을 찾을 수 있다 |

#### Should Have
| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 그룹 투표 | 팀 메뉴 결정 | P2 |
| 히스토리 | 식사 기록 관리 | P2 |
| 알림 | 점심시간 전 푸시 | P3 |

#### Could Have (Phase 2+)
- 식당 예약 연동
- 기업용 식대 관리 대시보드
- 배달 앱 연동

### 5. 비즈니스 모델

#### 수익 모델 (3가지)
```
1. 프리미엄 구독 (B2C)
   - Free: 일 3회 추천
   - Premium: 9,900원/월 (무제한 + 그룹 기능)

2. 기업 구독 (B2B)
   - Team: 5만원/월 (20인 이하)
   - Enterprise: 협의 (20인 초과 + 식대 관리)

3. 제휴 수수료
   - 예약 수수료: 5%
   - 배달 연동 수수료: 3%
```

#### 수익 예측 (1년차)
| 항목 | 월 예상 | 연 예상 |
|------|---------|---------|
| B2C Premium | 250만 | 3,000만 |
| B2B Team | 250만 | 3,000만 |
| 제휴 수수료 | 150만 | 1,800만 |
| **합계** | **650만** | **7,800만** |

### 6. 기술 스택 및 MVP

#### 기술 스택
- **Frontend**: React Native (iOS/Android 동시)
- **Backend**: Node.js + Express
- **DB**: PostgreSQL + Redis
- **AI**: OpenAI API → 자체 모델 (Phase 2)
- **Infra**: AWS (EC2, RDS, S3)

#### MVP 범위 (3개월)
- 소셜 로그인 (카카오)
- 선호도 설정 (5개 항목)
- AI 추천 (OpenAI API)
- 식당 지도 (카카오맵 연동)

### 7. 일정 및 마일스톤

| 주차 | 마일스톤 | 산출물 |
|------|----------|--------|
| 1-2주 | 기획 확정 | PRD, 와이어프레임 |
| 3-4주 | 디자인 완료 | UI/UX 시안 |
| 5-8주 | 개발 | 앱 + 서버 구현 |
| 9-10주 | QA | 테스트 리포트 |
| 11-12주 | 출시 | 앱스토어 등록 |

### 8. 리스크 및 대응

| 리스크 | 영향 | 확률 | 대응 |
|--------|------|------|------|
| AI 추천 정확도 낮음 | 높음 | 중 | 규칙 기반 하이브리드 |
| 사용자 확보 어려움 | 높음 | 중 | 기업 B2B 먼저 공략 |
| 경쟁사 진입 | 중 | 높음 | 그룹 기능으로 차별화 |

### 9. 기대 효과 및 KPI

| 지표 | 1개월 | 3개월 | 12개월 |
|------|-------|-------|--------|
| MAU | 1,000 | 10,000 | 100,000 |
| DAU | 300 | 3,000 | 30,000 |
| 전환율 | 3% | 5% | 10% |
| NPS | 40 | 50 | 60 |

## 예시 2: FocusFlow (집중력 관리 앱)

### 1. 프로젝트 개요
- **프로젝트명**: FocusFlow
- **한줄 요약**: 뽀모도로 + AI 분석 기반 집중력 관리 앱
- **유형**: 신규 개발
- **예상 기간**: 4개월

### 2. 문제 정의 및 해결책

#### 현황 분석
- 재택/하이브리드 근무 확산으로 자기 관리 어려움 증가
- 평균 집중 시간 11분 (Microsoft 2024 연구)
- 기존 뽀모도로 앱: 단순 타이머, 인사이트 부족

#### Pain Point
1. 집중이 언제 깨지는지 모름
2. 어떤 업무에 얼마나 시간 썼는지 파악 안 됨
3. 습관 개선을 위한 피드백 없음

#### 해결책
- **스마트 타이머**: 집중/휴식 자동 감지
- **생산성 분석**: 시간별/업무별 집중도 리포트
- **AI 코칭**: 개인 패턴 분석 후 최적 루틴 제안

### 3. 타겟 사용자

#### Primary: 프리랜서/1인 기업
- 자기 관리 필수, 시간 = 돈
- 생산성 도구에 적극 투자

#### Secondary: 재택 근무 직장인
- 집에서 집중하기 어려움
- 업무 시간 증명 필요성

### 4. 핵심 기능

#### Must Have
1. 뽀모도로 타이머 (25분/5분)
2. 업무 태그 (업무 유형 분류)
3. 일간/주간 리포트
4. 집중 모드 (알림 차단)

#### Should Have
5. AI 루틴 추천
6. 팀 대시보드
7. 캘린더 연동

### 5. 비즈니스 모델

```
1. 프리미엄 구독
   - Free: 기본 타이머
   - Pro: 4,900원/월 (분석 + AI 코칭)

2. 팀 플랜
   - 인당 3,000원/월 (팀 대시보드)

3. 기업 API
   - 생산성 데이터 API 제공 (협의)
```

### 6. 기술 스택

- **Frontend**: Flutter (iOS/Android/Web)
- **Backend**: Python FastAPI
- **DB**: MongoDB + InfluxDB (시계열)
- **AI**: 자체 ML 모델 (집중 패턴 분석)

### 7. KPI

| 지표 | 3개월 | 12개월 |
|------|-------|--------|
| DAU | 5,000 | 50,000 |
| 평균 세션 | 3회/일 | 5회/일 |
| Pro 전환 | 5% | 12% |

## 예시 3: PetCare (반려동물 건강 관리)

### 1. 프로젝트 개요
- **프로젝트명**: PetCare
- **한줄 요약**: 반려동물 건강 기록 + AI 이상 징후 감지
- **유형**: 신규 개발
- **예상 기간**: 5개월

### 2. 문제 정의

#### 현황
- 국내 반려동물 1,500만 마리 (2024)
- 동물병원 비용 부담 (진료비 평균 15만원)
- 초기 이상 징후 놓쳐서 악화되는 케이스 다수

#### Pain Point
1. 반려동물은 아프다고 말 못함
2. 언제 병원 가야 할지 판단 어려움
3. 건강 기록 관리 번거로움

#### 해결책
- **건강 다이어리**: 식사/배변/활동량 기록
- **AI 분석**: 패턴 이상 시 알림
- **병원 연결**: 증상 기반 적합 병원 추천

### 3. 타겟 사용자

#### Primary: 반려동물 초보 집사
- 첫 반려동물, 건강 관리 불안
- 정보 탐색에 적극적

#### Secondary: 다견/다묘 가정
- 여러 마리 관리 효율화 필요

### 4. 핵심 기능

#### Must Have
1. 반려동물 프로필 등록
2. 건강 기록 (식사/배변/체중)
3. 예방접종 일정 알림
4. 증상 체크리스트

#### Should Have
5. AI 이상 감지 알림
6. 병원 검색/예약
7. 커뮤니티

### 5. 비즈니스 모델

```
1. 프리미엄 구독
   - Free: 기본 기록
   - Premium: 6,900원/월 (AI 분석 + 무제한 반려동물)

2. 병원 제휴
   - 예약 수수료 10%
   - 광고 배너 (월 정액)

3. 쇼핑몰 연동
   - 사료/용품 판매 수수료 5%
```

### 6. KPI

| 지표 | 3개월 | 12개월 |
|------|-------|--------|
| 등록 반려동물 | 10,000 | 100,000 |
| MAU | 5,000 | 80,000 |
| 병원 예약 | 500건 | 10,000건 |

## 좋은 기획서의 공통점

### 1. 숫자로 말한다
- "많은 사용자" ❌ → "MAU 10만" ✅
- "빠른 결정" ❌ → "15분 → 1분" ✅

### 2. 문제-해결 연결이 명확하다
- 문제: 메뉴 결정 15분
- 해결: AI 추천으로 1분
- 효과: 결정 피로 93% 감소

### 3. BM이 다각화되어 있다
- 최소 2개 이상 수익원
- B2C + B2B 조합
- 구독 + 거래 수수료

### 4. MVP가 현실적이다
- 3개월 내 출시 가능한 범위
- Must Have 4-5개로 제한
- AI는 외부 API로 시작

### 5. 리스크를 인정한다
- 낙관적 전망만 하지 않음
- 구체적 대응 방안 포함
- 실패 시나리오도 고려
