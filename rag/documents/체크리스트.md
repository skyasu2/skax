# 웹/앱 기획서 검토 체크리스트

## 1. 논리성 검토 (40% 가중치)

### 1.1 문제 정의
- [ ] 해결하려는 문제가 명확하게 정의되었는가?
- [ ] 문제의 심각성이 데이터로 뒷받침되는가?
- [ ] "왜 이것이 문제인가?"에 대한 답이 설득력 있는가?
- [ ] 문제가 실제로 타겟 사용자에게 존재하는가?

### 1.2 해결책 연결
- [ ] 제안된 해결책이 문제를 직접 해결하는가?
- [ ] 문제 → 해결책 → 효과의 인과관계가 명확한가?
- [ ] "왜 이 방법인가?"에 대한 근거가 있는가?
- [ ] 다른 해결책 대신 이것을 선택한 이유가 있는가?

### 1.3 논리적 비약 검사
- [ ] 근거 없는 주장이 없는가?
- [ ] "~할 것이다" 표현에 합리적 근거가 있는가?
- [ ] 낙관적 가정만 있고 위험 요소를 무시하지 않았는가?
- [ ] 경쟁사 대비 차별점의 근거가 명확한가?

---

## 2. 실현 가능성 검토 (30% 가중치)

### 2.1 기술적 실현 가능성
- [ ] 제안된 기술 스택으로 핵심 기능 구현이 가능한가?
- [ ] AI/ML이 핵심이라면 구현 계획이 구체적인가?
- [ ] 외부 API 의존도가 적절한가? (단일 장애점 없음)
- [ ] 개발 팀의 역량으로 구현 가능한 수준인가?

### 2.2 MVP 범위
- [ ] MVP 기능이 3개월 내 출시 가능한 수준인가?
- [ ] Must Have 기능이 5개 이하인가?
- [ ] 핵심 가치를 전달하기에 MVP가 충분한가?
- [ ] Phase 2 이상으로 미룰 수 있는 기능이 MVP에 없는가?

### 2.3 비즈니스 모델
- [ ] 수익 모델이 2개 이상으로 다각화되었는가?
- [ ] "광고" 단일 BM이 아닌가?
- [ ] 가격 정책이 시장 수준에서 현실적인가?
- [ ] 수익 예측의 전환율/성장률이 현실적인가? (전환율 50% 이상은 비현실적)

### 2.4 일정 및 리소스
- [ ] 마일스톤별 일정이 현실적인가?
- [ ] 버퍼 시간이 포함되어 있는가?
- [ ] 필요 인력과 예산이 명시되었는가?
- [ ] 외부 의존성(API, 제휴 등)의 리드타임이 고려되었는가?

---

## 3. 구체성 검토 (20% 가중치)

### 3.1 정량적 목표
- [ ] 모든 목표가 숫자로 표현되었는가?
- [ ] MAU, DAU, 전환율 등 핵심 KPI가 명시되었는가?
- [ ] 목표 달성 시점이 명확한가? (1개월, 3개월, 12개월)
- [ ] 측정 방법이 정의되었는가?

### 3.2 타겟 사용자
- [ ] 페르소나가 구체적인가? (이름, 나이, 상황)
- [ ] "모든 사람"이 아닌 특정 그룹이 타겟인가?
- [ ] 사용자 니즈와 Pain Point가 구체적인가?
- [ ] 시장 규모 추정(TAM/SAM/SOM)이 있는가?

### 3.3 기능 명세
- [ ] 기능별 User Story가 있는가?
- [ ] 기능 우선순위(MoSCoW)가 명확한가?
- [ ] 개발 착수 가능한 수준의 상세 명세인가?
- [ ] UI/UX 흐름이 설명되었는가?

---

## 4. 완성도 검토 (10% 가중치)

### 4.1 필수 섹션 포함 여부
- [ ] 프로젝트 개요
- [ ] 문제 정의 및 해결책
- [ ] 타겟 사용자
- [ ] 핵심 기능
- [ ] 비즈니스 모델
- [ ] 기술 스택 및 MVP
- [ ] 일정 및 마일스톤
- [ ] 리스크 및 대응 방안
- [ ] 기대 효과 및 KPI

### 4.2 리스크 분석
- [ ] 기술 리스크가 식별되었는가?
- [ ] 비즈니스 리스크가 식별되었는가?
- [ ] 운영 리스크가 식별되었는가?
- [ ] 각 리스크에 대한 대응 방안이 있는가?

### 4.3 문서 품질
- [ ] 오타나 문법 오류가 없는가?
- [ ] 용어 사용이 일관적인가?
- [ ] 구조가 논리적으로 정리되었는가?
- [ ] 불필요한 반복이 없는가?

---

## 5. 치명적 문제 체크리스트 (FAIL 조건)

다음 중 하나라도 해당되면 전면 수정 필요:

### 5.1 비즈니스 모델 문제
- [ ] BM이 "광고" 하나뿐인가?
- [ ] 수익 모델이 전혀 없는가?
- [ ] 수익 예측이 비현실적인가? (전환율 50% 이상)

### 5.2 MVP 범위 문제
- [ ] MVP에 10개 이상 기능이 포함되었는가?
- [ ] AI/ML이 핵심인데 구현 계획이 없는가?
- [ ] 3개월 출시 목표인데 기능이 과도한가?

### 5.3 타겟 사용자 문제
- [ ] 타겟이 "모든 사람"인가?
- [ ] 페르소나가 전혀 없는가?
- [ ] 시장 규모 추정이 없는가?

### 5.4 논리성 문제
- [ ] 문제와 해결책이 연결되지 않는가?
- [ ] 경쟁사 대비 차별점이 없는가?
- [ ] 모든 주장에 근거가 없는가?

### 5.5 실현 가능성 문제
- [ ] 기술적으로 구현 불가능한 기능이 있는가?
- [ ] 일정이 비현실적인가?
- [ ] 필요 리소스가 과도한가?

## 6. 점수 산정 가이드

### 6.1 점수 기준
| 점수 | 상태 | 설명 |
|------|------|------|
| 9-10 | PASS | 완벽에 가까움, 바로 실행 가능 |
| 7-8 | REVISE | 양호함, 일부 보완 필요 |
| 5-6 | REVISE | 기본은 갖춤, 상당한 보완 필요 |
| 1-4 | FAIL | 치명적 문제, 전면 재설계 필요 |

### 6.2 감점 요소
| 문제 유형 | 감점 |
|-----------|------|
| 치명적 문제 (FAIL 조건) | -4점 이상 |
| 필수 섹션 누락 (개당) | -1점 |
| 구체적 수치 부재 | -1~2점 |
| 논리적 비약 | -1~2점 |
| MVP 과잉 | -2점 |
| BM 단일화 | -2점 |

### 6.3 가점 요소
| 우수 항목 | 가점 |
|-----------|------|
| 경쟁사 분석 우수 | +1점 |
| 데이터 기반 문제 정의 | +1점 |
| 구체적 리스크 대응 | +1점 |
| 상세한 기능 명세 | +1점 |

## 7. 개선 액션 아이템 예시

### 7.1 BM 다각화 필요 시
- B2C 프리미엄 구독 모델 추가
- B2B 기업/팀 플랜 추가
- 제휴 수수료 모델 추가
- 광고 외 2개 이상 수익원 확보

### 7.2 MVP 범위 축소 필요 시
- Must Have를 5개 이하로 제한
- AI 기능은 외부 API로 시작
- Phase 2로 미룰 수 있는 기능 분리
- 3개월 출시 가능 범위로 조정

### 7.3 구체성 보강 필요 시
- 모든 목표에 숫자 추가 (MAU, DAU, 전환율)
- 페르소나 상세화 (이름, 나이, 직업, 상황)
- 시장 규모 추정 (TAM/SAM/SOM)
- 기능별 User Story 작성

### 7.4 논리성 보강 필요 시
- 문제 정의에 데이터/출처 추가
- 해결책과 문제의 연결 강화
- 경쟁사 분석 및 차별점 명시
- 주장마다 근거 추가
